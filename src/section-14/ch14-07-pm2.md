# PM2

PM2 is a production process manager for Node.js. It handles clustering, monitoring, log management, and zero-downtime restarts. I use it on every production Express app because it makes operations so much easier than the raw cluster module.

## Installing PM2

```bash
npm install -g pm2
```

## Basic Usage

```bash
# Start your app with clustering (one process per CPU core)
pm2 start src/index.js -i max

# Start with a specific number of instances
pm2 start src/index.js -i 4

# Start without clustering (single instance)
pm2 start src/index.js
```

The `-i` flag enables cluster mode. `max` means one worker per CPU core.

## Ecosystem Config File

Instead of passing flags on the command line, I create an ecosystem file:

```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'my-express-app',
    script: 'src/index.js',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000,
    },
    env_development: {
      NODE_ENV: 'development',
      PORT: 3000,
    },
    max_memory_restart: '500M',
    error_file: './logs/error.log',
    out_file: './logs/out.log',
    merge_logs: true,
    log_date_format: 'YYYY-MM-DD HH:mm:ss Z',
  }],
};
```

Then:

```bash
pm2 start ecosystem.config.js
```

## Clustering with PM2

PM2 handles all the clustering logic for you:

```bash
# Scale up to 8 instances
pm2 scale my-express-app 8

# Scale down to 2 instances
pm2 scale my-express-app 2
```

You can scale up or down without restarting the app.

## Log Management

PM2 captures stdout and stderr from each process:

```bash
# View logs from all processes
pm2 logs

# View logs from a specific app
pm2 logs my-express-app

# Flush all logs
pm2 flush

# Install log rotation module
pm2 install pm2-logrotate
```

`pm2-logrotate` prevents log files from growing forever. It rotates them based on size or time.

## Monitoring

```bash
# See process status
pm2 list

# Detailed monitoring dashboard
pm2 monit

# Web-based monitoring
pm2 plus
```

The `pm2 monit` command shows a real-time dashboard with CPU and memory usage for each process:

```
┌──────────────────┐ ┌──────────────────┐
│ my-express-app-0 │ │ my-express-app-1 │
│  status: online  │ │  status: online  │
│  cpu: 12%        │ │  cpu: 8%         │
│  memory: 85MB    │ │  memory: 82MB    │
└──────────────────┘ └──────────────────┘
```

## Zero-Downtime Reload

```bash
pm2 reload ecosystem.config.js
```

`reload` restarts workers one at a time, so there is always at least one worker serving requests. This is different from `restart`, which kills all workers at once.

## Startup Script

To make PM2 start on system boot:

```bash
pm2 startup
pm2 save
```

This generates a startup script and saves the current process list. After a server reboot, PM2 automatically restores all your apps.

PM2 replaced my hand-rolled cluster setups and shell scripts. It is one of those tools that once you start using, you wonder how you managed without it.
