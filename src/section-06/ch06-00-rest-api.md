# Building REST APIs

Everything I have learned so far in Express leads to this: building real APIs. Routes, middleware, request handling, and response formatting all come together when you build a REST API. This is what most Express projects are used for, and honestly, it is the part I have been most excited to learn.

REST stands for Representational State Transfer. It is a set of principles for designing networked applications. You do not have to follow REST strictly, but understanding the principles helps you build APIs that are consistent, predictable, and easy for others to use.

In this section, I want to build a proper REST API from scratch. Not just endpoints that return JSON, but a well-designed API that follows conventions and best practices. Here is what I will cover:

- The core principles of REST and what they really mean
- How to design RESTful URLs and map HTTP methods
- Full CRUD operations with Express
- API versioning strategies
- Pagination and filtering for large datasets
- Sorting and field limiting for flexible queries
- Documenting your API with Swagger and OpenAPI
- Testing your API with Postman

I used to think building an API was just writing routes that return JSON. But there is so much more to it. Good API design makes the difference between an API people enjoy using and one that frustrates everyone.

Let me start by understanding what REST actually means.
